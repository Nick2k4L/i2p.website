I2P nyní vyžaduje Java 6 nebo vyšší. Důrazně doporučujeme aktualizaci na Java 7. Pokud stále používáte Java 5, musíte si aktualizovat Java před instalací I2P 0.9.12.

Verze 0.9.12 přidává předběžnou podporu pro ECDSA-podepsané Destinace. Obsahuje několik oprav pro zpracování zpráv o doručení (potvrzení) a tyto zprávy jsou nyní šifrovány na konci na konec pro vyšší bezpečnost.

Přešli jsme na Jetty 8. Jetty 8 je téměř identické s Jetty 7, takže nejsou potřeba složité převody konfiguračních souborů, jak tomu bylo u minulých aktualizací Jetty. Žádné ruční změny by neměly být nutné.

Jak je zvykem, doporučujeme aktualizaci na tuto verzi. Nejlepší způsob, jak zajistit bezpečnost a pomoci síti, je používat nejnovější verzi.

Na začátku března internetová vyhledávačská společnost [DuckDuckGo](https://duckduckgo.com/) [přispěla](https://duck.co/blog/foss2014) Invisible Internet Projectu (I2P) částkou 5000 dolarů v rámci svého ročního programu darování open-source projektům. Tato odměna byla udělena na základě [nominací](https://duck.co/forum/thread/5174/foss-donation-nomations-2014-edition) od veřejnosti na komunitním portálu DuckDuckGo.

Toto je největší dar, jaký I2P kdy obdržel, a odráží rostoucí zájem internetové komunity o soukromí a bezpečnost. Tato podpora pomůže I2P dosáhnout na více uživatelů, rozšířit vývoj a provést audit kódu. Umožní také vývojářům I2P účastnit se konferencí, jako je například konference [Real-World Cryptography](http://www.realworldcrypto.com/) v New Yorku, kde se vývojáři setkali a spolupracovali s odborníky na kryptografii ve shodě s cíli I2P poskytovat anonymitu veřejnosti.

I2P děkuje Gabrielu Weinbergovi a DuckDuckGo za štědrý dar a komunitě I2P za podporu při nominacích.

## PODROBNOSTI VERZE

### Vylepšení anonymity

- Šifrování zpráv o doručení
- Přidána předběžná podpora pro ECDSA-podepsané Destinace
- Přidána kontrola opakovaných požadavků na NTCP relaci
- Přidána omezení a blokování při zpracování streamovacích pingů

### Opravy chyb

- Oprava výměny RouterInfo v NTCP
- Prodloužení časového limitu pro zprávy o doručení
- Vyřazení streamovaných zpráv z nedávno uzavřených spojení
- Oprava restartů na Raspberry Pi
- Obnovení pokročilé konfigurace profileOrganizer.sameCountryBonus
- Oprava pro jwebcache a i2phex

### Ostatní

- Jetty 8.1.14.v20131031; nyní vyžadováno Java 6
- Opět snížen cílový počet spojení, aby se dále snížila míra odmítnutí tunelů
- Přidáno omezení rychlosti pro odchozí spojení na koncových bodech tunelů
- Přidána volitelná blokování inproxy v i2ptunnel
- Použití SSU relačního klíče pro požadavky/odpovědi při přenosu, pokud je k dispozici
- Zahrnutí dat HTTP POST do SYN paketu
- Přidána knihovna getopt pro lepší zpracování argumentů
- Další odstraňování závislostí na Jetty
- Odstraněn kód MD5, místo toho použity knihovny Java
- Změna výchozí adresy odběru adresáře
- Aktualizace překladů
- Aktualizace dat GeoIP (pouze nové instalace a PPA)
