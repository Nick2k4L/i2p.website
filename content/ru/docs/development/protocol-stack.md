Стек I2P представляет собой многоуровневую архитектуру, обеспечивающую анонимную связь. Каждый уровень добавляет специфические возможности поверх нижележащих. Смотрите [Индекс технической документации](/docs/develop/overview) для получения дополнительной информации о каждом компоненте.

## Уровень Интернета {#internet}

**IP** - Internet Protocol позволяет адресовать хосты в обычном интернете и маршрутизировать пакеты через интернет с использованием доставки по принципу "наилучшие усилия".

## Транспортный уровень {#transport}

- **TCP** - Протокол управления передачей обеспечивает надежную доставку пакетов в правильном порядке
- **UDP** - Протокол пользовательских датаграмм обеспечивает ненадежную доставку пакетов без гарантии порядка

## Транспортный уровень I2P {#i2p-transport}

Зашифрованные соединения между router'ами (пока не анонимные):

- **[NTCP2](/docs/specs/ntcp2)** - TCP транспорт на основе NIO
- **[SSU2](/docs/specs/ssu2)** - Безопасный полунадёжный UDP транспорт

## Уровень туннелей I2P {#tunnels}

Обеспечивает полностью анонимные зашифрованные tunnel соединения:

- **[Tunnel сообщения](/docs/legacy/tunnel-message)** - Зашифрованные I2NP сообщения и зашифрованные
  инструкции для их доставки
- **[I2NP сообщения](/docs/specs/i2np)** - Протокольные сообщения со слоистым шифрованием для
  многоузлового анонимного маршрутизации

## Уровень Garlic в I2P {#garlic}

Обеспечивает зашифрованную и анонимную доставку I2P-сообщений от точки к точке:

- **[Garlic messages](/docs/overview/garlic-routing)** - Обёрнутые I2NP сообщения для анонимной доставки

## I2P клиентский уровень {#client}

- **[I2CP](/docs/specs/i2cp)** - I2P Control Protocol позволяет приложениям получать доступ к сети I2P без необходимости использовать API router напрямую

## Транспортный уровень I2P End-to-End {#e2e-transport}

- **[Streaming Library](/docs/api/streaming)** - Обеспечивает надежную доставку в правильном порядке, аналогично TCP
- **[Datagram Library](/docs/api/datagrams)** - Обеспечивает ненадежную доставку, аналогично UDP

## Слой интерфейса приложений I2P {#app-interface}

Дополнительные интерфейсы для разработчиков приложений:

- **[I2PTunnel](/docs/api/i2ptunnel)** - Туннелирует TCP-соединения в I2P и из I2P
- **[SAMv3](/docs/api/samv3)** - Протокол Simple Anonymous Messaging для приложений, не использующих Java

## Прокси-слой приложений I2P {#app-proxy}

Прокси для стандартных интернет-протоколов:

- **HTTP** - Прокси для веб-браузинга
- **IRC** - Прокси для Internet Relay Chat
- **[SOCKS](/docs/api/socks)** - SOCKS4/4a/5 прокси
- **Streamr** - UDP прокси для потокового вещания

## Приложения {#applications}

Приложения могут взаимодействовать с I2P на различных уровнях:

**Streaming/Datagram приложения:** - I2P-совместимые приложения, использующие streaming или datagram библиотеки напрямую

**SAM-приложения:** - Приложения на любом языке, использующие протокол SAM

**Приложения специально для I2P:** - Приложения, разработанные специально для I2P (I2PSnark, SusiMail и т.д.)

**Стандартные интернет-приложения:** - Обычные приложения, использующие I2P прокси (веб-браузеры, IRC клиенты и т.д.)
